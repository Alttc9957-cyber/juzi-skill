# AI Agent 进化工具包 v2.0 — 使用指南

> 这份指南帮你快速上手 Agent 进化包。
> 10 分钟读完，按顺序喂给你的 Agent，让它自己进化。
> 版本：v2.0 | 更新日期：2026-07-09

---

## 你是谁？

- 用 AI Agent 做事的个人或小团队
- 感觉 AI "时好时坏"，不够稳定
- 想让 AI 越用越懂你，而不是每次从零开始
- 想省 token 但不想牺牲质量

## 这个工具包是什么？

一套**给 Agent 用的自我进化文档**。

不是给大模型聊天用的 prompt，而是让你的 Agent 读完之后，按文档里的方法一步步优化自己的能力结构。就像给一个新员工一套工作方法论，让他按方法论来做事。

**使用方式**：把文档一篇一篇喂给你的 Agent，每篇配合对应的提示词，Agent 会自己阅读、诊断、优化、验证。

## 喂给 Agent 的顺序

| 顺序 | 文档 | 为什么这个顺序 | 预计耗时 |
|------|------|--------------|---------|
| 1 | 进化机制 v2.0 | **先装发动机**。Agent 先学会怎么自我进化，后面的知识才能沉淀 | 10-15 分钟 |
| 2 | 记忆方法论 v2.0 | **再装硬盘**。进化产生的知识需要可靠的记忆系统来存 | 10-15 分钟 |
| 3 | 调研管线 v2.0 | **装雷达**。增强信息收集和处理能力 | 10-15 分钟 |
| 4 | 知识库架构 v2.0 | **建图书馆**。调研产出的知识需要结构化组织 | 10-15 分钟 |
| 5 | 路由决策树 v2.0 | **装调度员**。优化每次任务的执行效率 | 10-15 分钟 |
| 6 | 思维框架 v2.0 | **装参谋**。提升决策质量（可选） | 10-15 分钟 |

**总计**：全部喂完约 1-1.5 小时，Agent 完成基础能力升级。

## 测试验证方法

每喂完一篇，用一个真实任务来测试 Agent 是否真的内化了：

| 喂完 | 测试任务 | 观察什么 |
|------|---------|---------|
| 进化机制 | 让 Agent 做一个项目级任务，然后观察它是否主动复盘 | 有没有三角验证？有没有结构化产出？ |
| 记忆体系 | 告诉 Agent 一个偏好，过几轮对话后再看它是否还记得 | 有没有写入记忆？后续任务有没有遵守？ |
| 调研SOP | 让 Agent 做一个调研任务 | 有没有先建假设？有没有多源验证？有没有标注置信度？ |
| 知识库 | 让 Agent 完成任务后沉淀知识 | 有没有过三问闸门？有没有建交叉链接？ |
| 路由框架 | 给一个模糊需求 | 有没有判断大类？有没有输出置信度？模糊时有没有澄清？ |
| 思维框架 | 让 Agent 做一个需要决策的任务 | 有没有结构化拆解？交付前有没有自检？ |

### 验收标准

- ✅ Agent 能主动执行文档中的流程，不需要你提醒
- ✅ Agent 能说出自己用了哪个机制/流程
- ✅ 同样的错误不犯第二次
- ✅ 交付质量有可观测的提升

## 常见问题

### Q：我的 AI 工具没有记忆功能怎么办？

A：用 Markdown 文件手动管理。每次对话开始时，把 MEMORY.md 的内容贴给 AI；对话结束后，把重要的东西写回文件。支持文件读写的工具（如 Claude Code、Cursor、Cola）效果最好。

### Q：路由功能我的工具不支持怎么办？

A：手动切换。在发消息前花 3 秒判断"这是闲聊还是复杂任务"，然后手动选模型。或者让 Agent 在每次任务开始时先输出自己的判断，你确认后再执行。

### Q：需要全部 6 篇都喂完才有用吗？

A：不需要。按优先级：
- **最小可用**：只喂 1（进化机制）+ 2（记忆体系），效果最明显
- **标准配置**：加上 3（调研SOP）+ 4（知识库），能力大幅提升
- **完整配置**：全部 6 篇，体系化运转

### Q：喂完之后 Agent 没有变化怎么办？

A：检查三件事：
1. Agent 是不是真的按"执行指南"做了？（看它的输出有没有诊断和优化动作）
2. 你有没有给真实的任务来验证？（纸上谈兵没用）
3. 有没有给反馈？（Agent 的进化靠你的反馈驱动）

### Q：这个包适合什么 AI 工具？

A：任何支持文件读写或记忆功能的 AI Agent 都适用：
- **最佳体验**：Claude Code、Cursor、Cola、Codex（有文件系统+记忆）
- **良好体验**：ChatGPT（手动贴文件）、Coze（有记忆功能）
- **基础体验**：任何能读 Markdown 的 Agent

### Q：v1.0 的用户需要重新喂吗？

A：建议重新喂 v2.0。v2.0 相比 v1.0 增加了大量实操细节、实证数据、模板和 Agent 执行指南，结构也有调整。直接覆盖即可。

### Q：喂的时候有什么注意事项？

A：
1. **一篇一篇喂**，不要一次性全丢。每篇之间让 Agent 完成优化动作
2. **用真实任务验证**，不要只让 Agent 说"我理解了"
3. **给反馈**。Agent 做对了说"可以"，做错了立即纠正——这是进化的燃料
4. **不要追求完美**。先把框架跑起来，细节边用边调
