## v2.1 核心升级：自举安装

### 新增模块
- **00_自举安装指南/自举安装_v2.1.md** — 让 AI 自动安装进化系统，无需手动逐篇喂食

### 解决了什么问题
- v2.0 需要用户按顺序逐篇喂食文件，AI 不会主动内化
- v2.1 AI 读取工具包后，自动将核心规则写入 MEMORY.md，成为永久行为规则
- 每次会话自动加载，无需手动提醒

### 如何使用
1. 将整个工具包文件夹发给 AI
2. 告诉它"读取 00_自举安装指南/自举安装_v2.1.md 并执行安装"
3. AI 会自动读取所有模块 → 提取核心规则 → 写入系统记忆 → 创建目录结构 → 验证安装
4. 安装完成后，进化机制自动运行

## 六大体系互相关系

```
            ┌─────────────┐
            │  进化体系    │ ← 任务完成后驱动复盘和沉淀
            │  (自进化)    │
            └──────┬──────┘
                   │ 驱动
        ┌──────────┼──────────┐
        ↓          ↓          ↓
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 记忆体系  │ │ 调研SOP  │ │ 知识库   │
│ (存储层)  │ │ (能力层) │ │ (沉淀层) │
└──────────┘ └──────────┘ └──────────┘
        ↑          ↑          ↑
        └──────────┼──────────┘
                   │ 支撑
            ┌──────┴──────┐
            │ 路由框架    │ ← 任务开始时选择最优路径
            │ (效率层)    │
            └─────────────┘
                   │
            ┌──────┴──────┐
            │ 思维工具箱  │ ← 贯穿所有决策环节
            │ (方法论层)  │
            └─────────────┘
```

- **进化体系**是发动机——驱动整个系统持续运转
- **记忆体系**是硬盘——管理信息的写入、存储和检索
- **调研SOP**是雷达——在需要信息收集时启动八阶段管线
- **知识库**是图书馆——管理沉淀的知识，定期审核和维护
- **路由框架**是调度员——在每个任务开始时选择最优路径
- **思维工具箱**是参谋——在所有决策环节提供方法论支撑

## 核心原则（10 条铁律）

1. **做中学，不是学中做** — 能力在真实项目中提升，不是看资料
2. **用户反馈是最高质量的进化信号** — 比任何自我反思都管用
3. **宁可少而精，不要多而水** — 知识库 20 篇 A 级好过 100 篇水货
4. **可观测才能进步** — 没有指标的进化是自我安慰
5. **机制比意志重要** — 靠"我要努力"坚持不下来，靠机制才能持续
6. **分层记忆，重点突出** — 不是记得越多越好，而是重要的不能丢
7. **假设先行，多源验证** — 不漫无目的搜信息，不轻信单一来源
8. **按擅长领域路由** — 不是贵的好，是合适的好
9. **失效不删，保留可追溯** — 只标记不删除
10. **同坑不踩第二次** — 工具经验、踩坑教训必须沉淀

## v2.0 → v2.1 升级说明

| 维度 | v2.0 | v2.1 |
|------|------|------|
| 安装方式 | 手动逐篇喂食 | 自动读取+自举安装 |
| 规则存储 | 外部文档 | MEMORY.md永久规则 |
| 触发方式 | 需要提醒 | 自动触发 |
| 持续性 | 每次会话需重新加载 | 一次安装，永久生效 |
| 可观测性 | 无 | 进化日志+指标追踪 |

### 新增内容
- **00_自举安装指南** — 完整的自举安装流程，让 AI 自动将进化规则写入系统记忆
- **自动触发机制** — 用户纠正时、项目完成时、重复操作时、会话结束时，自动触发对应进化机制
- **进化日志模板** — 结构化记录用户纠正、成功模式、失败教训、防御机制、指标追踪
