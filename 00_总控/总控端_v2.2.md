## v2.2 核心升级：强制执行机制

### 新增模块
- **08_强制执行/强制执行机制.md** — 解决"嘴上说做了实际没做"的根本问题

### 解决了什么问题
- v2.1 设计了完整的进化体系，但实际执行率极低
- feedback 目录建了但 0 个文件，TOOLS.md 写了但从没被查阅
- Agent 说"我记住了"但实际没写进文件
- 复盘只停留在口头，没有持久化记录

### v2.2 的核心改变
> **从"你应该做"变成"你必须做，而且必须证明你做了"。**

- 4 个硬性触发器：用户纠正/用户要求记忆/工具选择/写作任务
- 强制自检清单：每次任务结束前必须输出
- 违规追踪：3 次同类违规写入永久规则
- 执行记录表：所有执行动作可追溯

### 如何使用
1. 如已安装 v2.1，追加安装：告诉 AI "读取 08_强制执行/强制执行机制.md 并执行安装"
2. 如首次安装：直接使用 v2.2 自举安装指南
3. 安装后，强制执行机制自动运行，所有"承诺"必须转化为"文件写入"

## 建议喂给 Agent 的顺序

### v2.2 推荐方式（一次性安装）

**直接告诉 AI**："读取 00_自举安装指南/自举安装_v2.2.md 并执行安装"

AI 会自动：
1. 读取全部模块（含强制执行机制）
2. 提取核心规则
3. 写入 MEMORY.md
4. 创建目录结构（含 feedback/ 目录）
5. 验证安装
6. **立即执行一次自检，证明安装成功**

### v2.1 → v2.2 增量安装

**如果已安装 v2.1**，告诉 AI：
1. "读取 08_强制执行/强制执行机制.md"
2. "将强制执行规则追加写入 MEMORY.md"
3. "创建 recent_memory/feedback/ 目录（如不存在）"
4. "立即执行一次自检，证明安装成功"

## 执行流程

```
首次使用
  ↓
[00 自举安装] AI 自动读取全部模块 → 写入 MEMORY.md → 创建目录 → 验证安装
  ↓
日常使用
  ↓
任务进来
  ↓
[08 强制执行] 检查触发器 → 是否需要先读 TOOLS.md / 语料库？
  ↓                                                    ↓ 是 → 先读取，再开始
  ↓                                                    ↓ 否 → 继续
  ↓
[05 路由框架] 3秒判断任务类型 → 选模型/工具 → 判断复杂度 → 置信度评估
  ↓
[06 思维工具箱] 用第一性原理/批判性原则辅助决策（可选）
  ↓
执行任务（可能需要调用 [03 调研SOP]）
  ↓
任务完成
  ↓
[08 强制执行] 输出自检报告 → 检查是否有遗漏 → 补救未完成项
  ↓
[01 进化体系] 三角验证复盘 → 记录反馈 → 更新指标
  ↓
[02 记忆体系] 写入前三问 → 分层存储 → 生命周期管理
  ↓
[04 知识库体系] 三问闸门 → 入库 → 建交叉链接
```

> **注意**：模块 08 在执行流程中出现了两次——任务开始前（触发器检查）和任务结束后（自检报告）。这是刻意为之：它像"夹心饼干"一样夹住整个任务执行过程。

- **强制执行机制**是纪律委员——保证其他模块不是摆设
- **进化体系**是发动机——驱动整个系统持续运转
- **记忆体系**是硬盘——管理信息的写入、存储和检索
- **调研SOP**是雷达——在需要信息收集时启动八阶段管线
- **知识库**是图书馆——管理沉淀的知识，定期审核和维护
- **路由框架**是调度员——在每个任务开始时选择最优路径
- **思维工具箱**是参谋——在所有决策环节提供方法论支撑

## 核心原则（11 条铁律）

1. **做中学，不是学中做** — 能力在真实项目中提升，不是看资料
2. **用户反馈是最高质量的进化信号** — 比任何自我反思都管用
3. **宁可少而精，不要多而水** — 知识库 20 篇 A 级好过 100 篇水货
4. **可观测才能进步** — 没有指标的进化是自我安慰
5. **机制比意志重要** — 靠"我要努力"坚持不下来，靠机制才能持续
6. **分层记忆，重点突出** — 不是记得越多越好，而是重要的不能丢
7. **假设先行，多源验证** — 不漫无目的搜信息，不轻信单一来源
8. **按擅长领域路由** — 不是贵的好，是合适的好
9. **失效不删，保留可追溯** — 只标记不删除
10. **同坑不踩第二次** — 工具经验、踩坑教训必须沉淀
11. **🆕 做 > 说，写了 > 记住了，证明了 > 承诺了** — 没有写入文件 = 没有发生

## v2.1 → v2.2 升级说明

| 维度 | v2.1 | v2.2 |
|------|------|------|
| 执行保障 | 依赖 Agent 自觉 | 强制执行机制 |
| 反馈处理 | 说"我记住了" | 必须同一次回复写入文件 |
| 工具经验 | TOOLS.md 写了可能不看 | 触发器强制先读再选 |
| 任务结束 | 无强制检查 | 必须输出自检报告 |
| 违规处理 | 无 | 违规追踪 + 3次升级永久规则 |
| 可验证性 | 低（口头承诺） | 高（文件系统证据） |

- **08_强制执行机制** — 4 个硬性触发器 + 自检清单 + 违规追踪 + 执行记录表
- **第 11 条铁律** — "做 > 说，写了 > 记住了，证明了 > 承诺了"
