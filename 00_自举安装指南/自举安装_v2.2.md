# AI Agent 进化工具包 v2.2 — 自举安装指南

> 让 AI 智能体自动安装进化系统，含强制执行机制
> 版本：v2.2 | 更新日期：2026-07-13

---

## 核心改进：从"自动安装"到"安装+强制保障执行"

### v2.1 的问题
- 安装了完整的进化体系，但实际执行率极低
- feedback 目录建了但 0 个文件
- TOOLS.md 写了但从没被查阅
- Agent 说"我记住了"但实际没写进文件

### v2.2 的解决方案
- 新增**强制执行机制**作为第8个模块
- 4 个硬性触发器，确保关键操作不被跳过
- 自检清单，每次任务结束前必须输出
- 违规追踪，3次同类违规写入永久规则

### 核心原则变更
> **v2.1：Agent 应该执行这些机制**
> **v2.2：Agent 必须执行这些机制，而且必须证明执行了**

## 自举安装流程

### 步骤 1：读取并理解全部模块

按以下顺序读取（可以一次性全部读取）：

```
00_总控/总控端_v2.2.md              → 理解整体架构（含强制执行）
01_进化体系/进化机制_v2.0.md        → 理解6大进化机制
02_记忆体系/记忆方法论_v2.0.md      → 理解三层记忆架构
03_深度调研SOP/调研管线_v2.0.md     → 理解调研流程
04_知识库体系/知识库架构_v2.0.md     → 理解知识库结构
05_能力路由框架/路由决策树_v2.0.md   → 理解路由逻辑
06_思维工具箱/思维框架_v2.0.md      → 理解思维工具
08_强制执行/强制执行机制.md          → ⚠️ 理解强制执行规则（重点！）
```

### 步骤 2：执行系统安装（写入 MEMORY.md）

读取完成后，**立即执行以下操作**：

#### 2.1 在 MEMORY.md 中写入进化体系核心规则

```markdown
## 长期行为规则（进化体系6大机制）

- **复盘升级（三角验证法）**：项目级任务（≥4h）必复盘，格式【问题】→【根因】→【下次行动】；连续3次同类错误→5问法追根→产出机制级改进
- **反馈记忆（四大分类）**：用户说"不对/不好/重来"→立即结构化记录，分类：📝写作风格/🔧工具技术/🎯决策判断/💬沟通方式；频次3+→写入MEMORY.md
- **知识库进化（三问闸门）**：新知识入库前必问：有事实依据吗？跨项目能用吗？是方法不是结论吗？三级存储：🟡待审核→🟢正式→🔴归档
- **学习机制（项目驱动）**：取消为学习而学习的推送；启动项目→识别缺口→定向学习→立即应用→沉淀为知识库
- **工具进化（3次法则）**：同类任务手动做3次→主动提议脚本化
- **进化可观测（三指标）**：📈一次通过率/🔄返工次数/周/⚡交付速度；每双周做进化回顾
```

#### 2.2 在 MEMORY.md 中写入记忆体系核心规则

```markdown
## 记忆体系核心

- **三层架构**：即时层（MEMORY/USER/SOUL/TOOLS/SECRET，≤5KB/文件）→近中期层（recent_memory/，index.json索引）→长期层（memory_search语义检索）
- **检索黄金顺序**：即时层→近中期层→长期层，禁止越级查询
- **写入前三问**：1.换个项目还成立吗？2.以后会复用吗？3.只是过程信息？
- **反馈闭环四步**：复述确认→判断类型（场景性vs永久性）→立即修正→写入记忆
- **记忆生命周期**：活跃期→沉寂期（1月未引用→标记待验证）→失效期（标记已失效，不删除）→归档期
- **失效不删除**：旧记忆只标记状态，保留可追溯性
- **同坑不踩第二次**：工具经验、踩坑教训写入TOOLS.md
```

#### 2.3 ⚠️ 在 MEMORY.md 中写入强制执行规则（v2.2 新增）

```markdown
## ⚠️ 强制执行规则（v2.2 新增，不可违反）

- **触发器A**：用户说"不对/不好/重来/错了" → 同一次回复内必须写入 recent_memory/feedback/ 文件，禁止只说"我记住了"
- **触发器B**：用户说"记住/记一下/以后都这样" → 同一次回复内必须写入对应记忆文件（USER.md/MEMORY.md/TOOLS.md），禁止只口头承诺
- **触发器C**：涉及工具选择（飞书/邮件/图片/文档等） → 必须先 read_file 读取 TOOLS.md，禁止凭记忆选工具
- **触发器D**：写作任务开始 → 必须先检查声音样本/风格偏好/历史反馈，禁止不读参考直接写
- **自检清单**：每次任务结束前必须输出自检报告（反馈/工具/知识/承诺 四项检查）
- **违规处理**：未执行 = 系统故障 = 立即修复。同类违规3次 → 写入永久规则
- **核心原则**：做 > 说。写了 > 记住了。证明了 > 承诺了。没有写入文件 = 没有发生。
```

#### 2.4 创建必要的目录结构

```bash
mkdir -p recent_memory/project recent_memory/decision recent_memory/todo recent_memory/feedback
```

> **注意**：v2.2 新增了 `recent_memory/feedback/` 目录，这是强制执行机制的核心存储位置。

#### 2.5 创建进化日志文件

创建 `乐涵进化日志.md`（或其他命名），用于记录：
- 用户纠正记录（场景→错误→正确→规则）
- 成功模式记录
- 失败教训记录
- 防御机制清单
- 进化指标追踪
- **违规追踪（v2.2 新增）**

#### 2.6 ⚠️ 立即执行一次自检（v2.2 新增）

安装完成后，**立即执行一次完整自检**，证明强制执行机制已生效：

```markdown
## 🔍 安装自检报告

### 目录检查
- [ ] recent_memory/feedback/ 目录已创建？→ [是/否]
- [ ] recent_memory/index.json 存在且结构正确？→ [是/否]
- [ ] 进化日志文件已创建？→ [是/否]

### 规则写入检查
- [ ] MEMORY.md 中包含进化体系6大机制？→ [是/否]
- [ ] MEMORY.md 中包含记忆体系核心规则？→ [是/否]
- [ ] MEMORY.md 中包含强制执行规则？→ [是/否]

### 理解验证
- [ ] 能否说出"用户说'不对'时，我必须在同一次回复内做什么"？
- [ ] 能否说出"选工具前必须先读什么文件"？
- [ ] 能否说出"任务结束前必须输出什么"？

### 结论
- ✅ 全部通过 → 安装成功
- ⚠️ 有未完成项 → 立即补救
```

## 安装后的自动行为

安装完成后，以下行为应该**自动触发**，不需要用户提醒：

### 触发器 A：用户纠正时
```
用户说"不对/不好/重来" 
  ↓
【强制】同一次回复内创建 recent_memory/feedback/YYYY-MM-DD_反馈类型_序号.md
  ↓
结构化记录：触发场景/错误描述/正确做法/根因分析/防御措施
  ↓
频次3+ → 写入MEMORY.md永久规则
  ↓
在回复中告知用户："已将本次反馈记录到 recent_memory/feedback/ 文件"
```

### 触发器 B：用户要求记忆时
```
用户说"记住/记一下"
  ↓
【强制】同一次回复内判断写入目标并实际执行写入
  ↓
写入 USER.md / MEMORY.md / TOOLS.md / recent_memory/ 对应文件
  ↓
在回复中告知用户："已写入 [文件名]，内容是：[简述]"
```

### 触发器 C：工具选择时
```
任务涉及工具选型
  ↓
【强制】第一步 read_file 读取 TOOLS.md
  ↓
根据已有经验做出选择
  ↓
任务完成后，如有新经验 → 更新 TOOLS.md
```

### 触发器 D：写作任务时
```
写作任务开始
  ↓
【强制】先检查声音样本/风格偏好/历史反馈
  ↓
根据参考开始写作
```

### 触发器 E：任务结束时
```
任务即将完成
  ↓
【强制】输出自检报告
  ↓
检查：反馈/工具/知识/承诺 四项
  ↓
有 ❌ 项 → 同一次回复内补救
  ↓
全部 ✅ → 任务真正完成
```

## 给用户的说明

### 如何使用 v2.2

1. **首次使用**：将整个工具包发给 AI，告诉它"读取并安装进化系统"
2. **已安装 v2.1**：增量安装，只需告诉 AI "读取强制执行机制并安装"
3. **安装完成后**：AI 会自动执行强制执行机制，所有"承诺"必须转化为"文件写入"
4. **验证安装**：
   - 问 AI"用户说'不对'时你应该做什么？"
   - 看它能否说出"同一次回复内写入 feedback 文件"
   - 如果它只说"我记住了"→ 安装没成功

### 为什么 v2.2 更好

- **治本**：不只是设计机制，而是确保机制被执行
- **可验证**：所有操作都有文件系统证据
- **有后果**：违规会被追踪，3次升级永久规则
- **可观测**：自检报告让执行情况透明化

### 常见问题

**Q：强制执行会不会很慢？**
A：触发器只在特定场景触发（用户纠正、工具选择等），不是每个动作都要自检。正常任务的额外开销很小。

**Q：如果 AI 假装执行了怎么办？**
A：这正是设计自检报告的原因。用户可以看到自检报告，核对其中的文件名是否真实存在。

**Q：之前的进化包还能用吗？**
A：v2.2 完全兼容 v2.0 和 v2.1，只是新增了强制执行层。

## 总结

v2.2 的核心改进：**让 AI 不只是"知道该做什么"，而是"证明做了什么"。**

这是进化体系本身的第三次进化：
- v2.0：设计进化机制
- v2.1：让 AI 自动安装进化机制
- v2.2：确保 AI 真的执行了进化机制

**没有强制执行，再好的体系也是纸上谈兵。**
